# Excel文件格式说明

## 必需列

Excel文件必须包含以下5列（列名必须完全匹配）：

1. **表名** - 数据库表名称
2. **表含义** - 表的业务含义说明
3. **字段名** - 字段名称
4. **字段类型** - 字段数据类型（如：INT, VARCHAR, DATE等）
5. **字段含义** - 字段的业务含义说明

## 格式示例

| 表名 | 表含义 | 字段名 | 字段类型 | 字段含义 |
|------|--------|--------|----------|----------|
| users | 用户表 | id | INT | 用户唯一标识 |
| users | 用户表 | name | VARCHAR(100) | 用户姓名 |
| users | 用户表 | email | VARCHAR(255) | 用户邮箱地址 |
| users | 用户表 | created_at | DATETIME | 创建时间 |
| orders | 订单表 | id | INT | 订单唯一标识 |
| orders | 订单表 | user_id | INT | 用户ID（外键关联users表） |
| orders | 订单表 | amount | DECIMAL(10,2) | 订单金额 |
| orders | 订单表 | status | VARCHAR(20) | 订单状态（pending/paid/cancelled） |
| orders | 订单表 | created_at | DATETIME | 创建时间 |

## 注意事项

1. **列名必须准确**：列名必须完全匹配（区分大小写），包括中文字符
2. **表名一致性**：同一张表的"表名"列必须完全一致
3. **表含义一致性**：同一张表的"表含义"应该相同
4. **字段类型**：建议使用标准SQL数据类型，如：INT, VARCHAR, TEXT, DATE, DATETIME, DECIMAL等
5. **空值处理**：避免在关键字段（表名、字段名）中出现空值

## 使用建议

- **详细的表含义**：清晰描述表的业务用途，有助于大模型理解上下文
- **详细的字段含义**：说明字段的业务含义，特别是外键关系
- **数据类型信息**：包含长度信息（如VARCHAR(100)）有助于生成更准确的SQL
- **关系说明**：在字段含义中说明外键关系，如"用户ID（外键关联users表）"
