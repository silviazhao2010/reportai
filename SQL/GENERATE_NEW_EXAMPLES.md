# 生成全新问题和SQL的提示词指南

## 概述

本文档提供专门用于从数据库schema生成**完全不同的**新问题和SQL的提示词模板和最佳实践。

## 核心原则

### 1. 强调"完全不同"

在提示词中明确要求生成**完全不同的**问题，而不是相似或变体：

```markdown
### 关键要求（非常重要！）

1. **完全不同的查询**：
   - 每个问题必须查询不同的数据或使用不同的查询方式
   - 不要生成相似或重复的问题
   - 确保每个问题都有独特的查询意图
```

### 2. 覆盖不同的查询类型

明确列出需要覆盖的查询类型，确保多样性：

```markdown
2. **覆盖不同的查询类型**：
   - 必须包含以下查询类型（每种至少1个）：
     - 简单SELECT查询（单表）
     - 条件查询（WHERE）
     - 多表关联查询（JOIN）
     - 分组统计（GROUP BY）
     - 聚合函数（COUNT、SUM、AVG、MAX、MIN）
     - 排序查询（ORDER BY）
     - 子查询
     - 时间范围查询
     - 组合条件查询（AND/OR）
     - 限制结果数量（LIMIT）
```

### 3. 提供具体的示例

给出每个查询类型的具体示例，帮助模型理解：

```markdown
### 查询类型示例

#### 1. 简单查询
- "查询所有用户的姓名和邮箱"
- "列出所有订单信息"

#### 2. 条件查询
- "查询状态为活跃的用户"
- "查询金额大于1000的订单"

#### 3. 分组统计
- "统计每个部门的员工数量"
- "按日期统计每天的订单数量"
```

### 4. 使用分类策略

按类别分配问题数量，确保覆盖全面：

```markdown
### 生成策略（按类别分配）

请确保生成的30个问题覆盖以下类别：

#### 类别1：简单查询（5个）
#### 类别2：条件查询（5个）
#### 类别3：排序和限制（3个）
#### 类别4：分组统计（5个）
#### 类别5：聚合函数（4个）
#### 类别6：多表关联（4个）
#### 类别7：子查询（2个）
#### 类别8：时间查询（2个）
```

## 使用技巧

### 1. 温度参数设置

- **生成新问题**：使用较高的temperature（0.8-0.9）增加多样性
- **生成SQL**：使用较低的temperature（0.1-0.3）保持准确性

### 2. 分批次生成

不要一次性生成太多问题，建议：
- 每次生成10-20个
- 生成后检查质量
- 筛选出好的问题
- 再生成下一批

### 3. 迭代优化

根据生成结果调整提示词：
- 如果某些类型的问题太少 → 加强该类别的要求
- 如果问题太相似 → 加强"完全不同"的要求
- 如果SQL有错误 → 加强SQL语法要求

### 4. 使用示例引导

在提示词中提供好的示例：
- 展示不同查询类型的示例
- 展示不同复杂度的问题
- 展示不同的表达方式

### 5. 明确禁止事项

明确说明不要做什么：
- 不要生成相似的问题
- 不要重复使用相同的表和字段组合
- 不要生成无意义的查询
